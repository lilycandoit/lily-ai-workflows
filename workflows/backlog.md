# Backlog Workflow

Goal: capture ideas, seeds, bugs, research topics, and future work without disrupting current execution.

Use this when the user says to remember an idea, add something later, capture a seed, or put work in backlog.

## File

Use `.planning/BACKLOG.md`.

Create it if missing.

## Suggested Format

```md
# Backlog

## Ideas

## Bugs

## Future Features

## Research Needed

## Maybe Later
```

## Capture

For each item, include:

- title
- type: idea, bug, feature, research, chore, maybe-later
- source or context
- why it matters
- suggested phase, if obvious
- status: new

## Rules

- Do not change roadmap scope unless the user asks.
- Do not interrupt active plan execution.
- Keep entries short but useful.
- If the item is urgent or blocking, mention that it may belong in `STATE.md` blockers instead.

## Output

Return the added backlog item and file path.
